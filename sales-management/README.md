# 売上・在庫管理アプリ

レーザー機械の **入荷／在庫／出荷／納品先** を1台ごとに管理するデスクトップアプリ。

## 主な機能

- **📥 入荷登録**: 機種・シリアル・仕入先・金額を1台ごとに記録
- **シリアル重複警告**: 入力時に在庫中の同シリアル／過去履歴を即座に検知
- **📝 受注**: お客様情報・請求書・入金状態を先に登録、出荷時にシリアル割当
  - 「出荷確定」で在庫から選択 → 自動で売上計上＋出荷済へ
- **📋 一覧**: 全シリアルの状態色分け（在庫=黄／資産=緑／部品取り=水色／出荷済=灰）
  - 機種・シリアル・購入者で検索、列ヘッダクリックでソート、CSV書き出し
- **⚙ ツール**: 既存Excel取込、CSV書き出し、DBバックアップ
- **既存Excel取込**: 「全機種 シリアル照合表」シートを自動認識
  - 黄色／緑／水色マーカーから状態を自動判定
  - 結合セル展開、日付文字列パース、ロット列を仕入先として取り込み

## セットアップ

1. **Python 3.10 〜 3.13 をインストール**（**3.14 は Tkinter の不具合で動作不良が報告されています**。3.12 推奨）
   - https://www.python.org/downloads/windows/ から **Python 3.12.x** の Windows installer (64-bit) をダウンロード
   - インストール時に「**Add python.exe to PATH**」にチェック
2. このフォルダを任意の場所（例: デスクトップ）に展開
3. `run.bat` をダブルクリック → 初回は venv 作成と pip install が走ります
4. もしくは PowerShell から:
   ```
   cd "<展開先のパス>\sales-management"
   py -3.12 -m pip install openpyxl
   py -3.12 app.py
   ```

> ℹ️ **受注タブは完全インライン構成**：「＋新規受注」「📦出荷確定」を押すと、別ウィンドウ（ポップアップ）ではなく **同じ画面の下側にフォームが表示**されます。一覧の行を **ダブルクリック**（または行を選んで「編集」ボタン）で既存の受注を読み込みます。Windows + Tk のポップアップが固まる問題を避けるため、ポップアップを一切使わない設計です。Python 3.12 推奨。

## 使い方

### 初回：既存Excelの取込
1. アプリを起動
2. **「⚙ ツール」タブ → 「Excelファイルを選択してインポート」**
3. `シリアルNo.照合表YYYY.MM.DD.xlsx` を選択
4. 全行が DB に登録される（黄=在庫／緑=資産／水色=部品取り／白=出荷済）
5. **再取込時**: 既存データを削除するか追加するか確認ダイアログ

### 日々の運用

**入荷時（中国から数十台到着）**
- 「📥 入荷登録」で1台ずつ登録（または既存Excel再取込）
- シリアル重複は自動警告

**受注時（請求書発行〜入金）**
- 「📝 受注」→「＋ 新規受注」
- お客様情報・機種・請求書番号・金額を入力（シリアルはまだ未定）
- 入金確認後、状態を「入金済」に更新

**出荷時（シリアル確定）**
- 「📝 受注」→ 該当受注を選択 → 「📦 出荷確定」
- 在庫一覧から該当シリアルをクリック → 確定
- 自動で売上計上＋在庫から消える＋受注は「出荷済」に

**シリアルで履歴確認**
- 「📋 一覧」で検索欄にシリアル番号を入れる
- 行をダブルクリック → 入荷／出荷の全履歴表示

### シリアル重複防止
入荷登録でシリアル番号欄からフォーカスを外すと、以下が自動チェックされます：
- ⚠ **在庫中に同シリアルあり** → 重複入荷の警告（赤）
- ℹ **過去の出荷履歴あり** → 中古再販なら問題なしの注意（橙）
- ✓ 履歴なし → そのまま登録 OK

## ファイル構成

```
sales-management/
├── app.py             # Tkinter GUI（メイン）
├── db.py              # SQLite スキーマ
├── import_excel.py    # Excel 取込
├── export_csv.py      # CSV 書き出し
├── config.py          # パス設定
├── requirements.txt
├── requirements-dev.txt  # pytest 等の開発依存
├── run.bat            # Windows 起動用
├── tests/             # pytest テスト
└── data/              # ← 起動時に自動生成
    ├── sales.db       #   SQLite データベース本体
    ├── attachments/   #   添付画像/PDF
    └── exports/       #   CSV 書き出し先
```

## 開発者向け: テスト実行

```
py -3.12 -m pip install -r requirements-dev.txt
py -3.12 -m pytest
```

`tests/` は SQLite テンポラリDB を使うため、本番 `data/sales.db` には触れません。

## バックアップ

`data/sales.db` をコピーするだけで完全バックアップになります。アプリ内の
**「⚙ ツール → DBファイルをコピー保存」** からも実行可能。

## 既知の制約

- 機種名の表記揺れ（`LT6040-40w` / `LT6040-40w　MPC` 等）は自動正規化していません。
  検索は部分一致で動作します。
- 受注タブのフォームはインライン表示。一覧右上の「📦 出荷確定」ボタンを押した後、画面右側に在庫一覧 → クリックで割当 → 確定、の流れ。
- Python は 3.12 推奨。3.14 では他のごく一部の Toplevel ダイアログがハングするケースが報告されています。
