# 売上・在庫管理アプリ

レーザー機械の **入荷／在庫／出荷／納品先** を1台ごとに管理するデスクトップアプリ。

## 主な機能

- **入荷登録**: 機種・シリアル・仕入先・金額を記録
- **シリアル重複警告**: 入力時に在庫中の同シリアル／過去履歴を即座に検知
- **出荷登録**: 在庫から個体を選んで、お客様情報・納品先・金額を記録
- **一覧／検索**: お客様名・住所・機種・年で絞り込み、CSVへ書き出し
- **在庫サマリ**: 機種別の残数を一覧
- **詳細／画像**: 1台ごとに検品写真や伝票写真を紐付け（種別タグ付き）
- **既存Excel取込**: 過去の売上Excelをワンクリックで取り込み

## セットアップ

1. Python 3.10 以降をインストール
2. このフォルダを任意の場所（例: デスクトップ）に展開
3. `run.bat` をダブルクリック → 初回は venv 作成と pip install が走ります

## 使い方

### 初回：既存Excelの取込
1. `run.bat` でアプリを起動
2. **「⚙ ツール」タブ → 「Excelファイルを選択してインポート」**
3. `売上エクセル修正済YYYYMMDD.xlsx` を選択
4. 全行が DB に登録されます（status は `出荷済` 扱い）

### 日々の運用
- 機械が入荷したら **「📥 入荷登録」**
- 売れたら **「📤 出荷登録」** で在庫から個体を選び、お客様情報を入力
- 納品先が後で決まった場合は **「📋 一覧/検索」** で該当行をダブルクリック →「メモ編集」ボタンや CSV書き出しで管理
- 検品写真／伝票写真は **「🖼 詳細/画像」** タブから個体ID指定で添付

### シリアル重複防止
入荷登録でシリアル番号欄からフォーカスを外すと、以下が自動チェックされます：
- ⚠ **在庫中に同シリアルあり** → 重複入荷の警告（赤）
- ℹ **過去の出荷履歴あり** → 中古再販なら問題なしの注意（橙）
- ✓ 履歴なし → そのまま登録 OK

## ファイル構成

```
sales-management/
├── app.py             # Tkinter GUI（メイン）
├── db.py              # SQLite スキーマ
├── import_excel.py    # Excel 取込
├── export_csv.py      # CSV 書き出し
├── config.py          # パス設定
├── requirements.txt
├── run.bat            # Windows 起動用
└── data/              # ← 起動時に自動生成
    ├── sales.db       #   SQLite データベース本体
    ├── attachments/   #   添付画像/PDF
    └── exports/       #   CSV 書き出し先
```

## バックアップ

`data/sales.db` をコピーするだけで完全バックアップになります。アプリ内の
**「⚙ ツール → DBファイルをコピー保存」** からも実行可能。

## 既知の制約

- 既存Excelからの取込分は「入荷」情報なし（出荷のみ）。必要な場合は後から
  該当個体を選んで仕入先などを手入力してください。
- 機種名の表記揺れ（`LT6040-40w` / `LT6040-40w　MPC` 等）は自動正規化していません。
  検索は部分一致で動作します。
